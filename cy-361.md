## 项目主要需求

**项目概述**：实验室设备预约管理系统 —— 面向高校和科研院所，提供实验设备的在线预约、使用记录追踪、维护计划管理和权限分级控制，解决设备闲置与争抢并存的矛盾。

**项目类型**：全栈Web应用

**功能需求**：
1. **设备档案与分类管理**：支持录入设备名称、型号、编号、存放位置、购置日期、使用说明文档，可上传设备图片，支持按分类（如光学仪器、电子仪器、生化设备）筛选和检索。
2. **时段预约与资质校验**：设备开放预约的最小单位为30分钟，用户选择日期后查看该设备当日可约时段，预约时自动校验用户资质（如是否需要培训认证），冲突时段禁止预约。
3. **扫码签到与违约释放**：预约成功后生成使用二维码，使用人需在使用开始和结束时扫码签到，系统自动记录实际使用时长，超时未签到自动释放设备并标记违约。
4. **设备维护计划管理**：管理员可为设备设定定期维护周期（如每周/每月），维护时段自动从可预约池中剔除，维护完成后记录维护内容、耗材更换情况和维护人。
5. **利用率统计报表**：管理员可查看设备利用率报表（按周/月/年）、高频使用人排行、设备故障率统计，支持导出Excel格式的月度使用报告。

**技术要求**：
- 英文短名（用于 Docker Compose）：`ldequipreserve`
- 前端端口：`28501`，后端端口：`29501`
- 前端框架：React 18 + TypeScript
- UI 库：Ant Design
- 构建工具：Vite
- 后端框架：NestJS + TypeScript
- 数据库：MySQL 8.0
- 认证方式：JWT（Access Token + Refresh Token）
- 其他依赖：TypeORM、bcryptjs、class-validator

## 部署要求

项目必须支持 Docker Compose 一键部署，且在 **任意目录名（包括中文目录名）** 下都能正常启动。

需要提供以下文件：
- 根目录 `docker-compose.yml`：编排前端、后端、数据库及所有依赖服务
- 根目录 `.env`：包含 `COMPOSE_PROJECT_NAME=ldequipreserve`，确保 Compose 项目名不依赖目录名（中文目录名会被规范化为空导致 `project name must not be empty` 错误）
- `frontend/Dockerfile`：前端项目构建与运行镜像（多阶段构建，先构建再用 Nginx 托管）
- `backend/Dockerfile`：后端项目构建与运行镜像
- `frontend/nginx.conf`：Nginx 配置，处理前端路由和 API 反向代理
- `.env.example`：所有环境变量的示例文件，附注释说明；同时包含 `COMPOSE_PROJECT_NAME` 默认值

docker-compose.yml 要求：
- **不要写 `version:` 字段**（Compose V2 已废弃，会产生 `the attribute version is obsolete` 警告）
- 顶层声明 `name: ldequipreserve` 作为兜底，避免目录名为中文时项目名为空
- 所有服务使用 `container_name` 命名容器，便于管理；容器名建议带 `${COMPOSE_PROJECT_NAME:-ldequipreserve}` 前缀
- 数据库数据使用命名卷持久化（`volumes:` 顶层声明，不要使用绑定挂载到中文路径）
- 前端通过 Nginx 反向代理后端 API（前端代码里调用 `/api`，由 Nginx 转发到 `backend:29501`，不要硬编码 `localhost`）
- 服务间通过 Docker 内部网络通信，服务名即主机名
- 数据库服务配置 `healthcheck`，后端服务通过 `depends_on: { db: { condition: service_healthy } }` 等待数据库就绪
- 后端服务也建议配置 `healthcheck`，前端 `depends_on` 后端健康
- 暴露端口：前端 `80:80`，后端按框架默认端口，数据库使用标准端口
- 所有可变配置（端口、数据库账号密码、JWT 密钥等）通过环境变量注入，并在 `.env.example` 中给出默认值
- 支持通过 `docker compose up -d` 一条命令启动全部服务，不依赖额外的 `-p` 参数

`.env` / `.env.example` 必须包含的键：
- `COMPOSE_PROJECT_NAME`：英文短名（小写字母/数字/`-`/`_`），如 `ldequipreserve`
- 数据库连接相关：`DB_NAME` / `DB_USER` / `DB_PASSWORD` / `DB_ROOT_PASSWORD`
- 后端：`JWT_SECRET` 或等价的密钥配置
- 端口映射：`FRONTEND_PORT`、`BACKEND_PORT`、`DB_PORT`（可选，便于本地冲突时调整）

Nginx 反向代理要求（`frontend/nginx.conf`）：
- `location /api/` 反向代理到 `http://backend:29501/`
- 配置 `try_files $uri $uri/ /index.html;` 支持前端 SPA 路由
- WebSocket 场景需配置 `Upgrade` / `Connection` 头

自检清单（生成代码后必须确认）：
1. 在 **中文目录名** 下执行 `docker compose config --quiet` 不报错
2. 启动后 `docker compose ps` 显示全部容器 healthy
3. 浏览器访问前端端口能看到页面，且 `/api/...` 请求返回正常
4. README 中的 Docker 启动命令是 `docker compose up -d`（不需要 `-p`）

## docker-compose.yml 参考骨架（务必遵循）

```yaml
name: ldequipreserve   # 顶层 name 字段兜底，避免中文目录名导致 project name 为空

services:
  db:
    image: mysql:8.0
    container_name: ${COMPOSE_PROJECT_NAME:-ldequipreserve}-db
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASSWORD}
      MYSQL_DATABASE: ${DB_NAME}
      MYSQL_USER: ${DB_USER}
      MYSQL_PASSWORD: ${DB_PASSWORD}
    volumes:
      - db_data:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-u", "root", "-p${DB_ROOT_PASSWORD}"]
      interval: 5s
      timeout: 5s
      retries: 20

  backend:
    build: ./backend
    container_name: ${COMPOSE_PROJECT_NAME:-ldequipreserve}-backend
    restart: unless-stopped
    environment:
      DB_HOST: db
      DB_PORT: 3306
      DB_NAME: ${DB_NAME}
      DB_USER: ${DB_USER}
      DB_PASSWORD: ${DB_PASSWORD}
      JWT_SECRET: ${JWT_SECRET}
    depends_on:
      db:
        condition: service_healthy

  frontend:
    build: ./frontend
    container_name: ${COMPOSE_PROJECT_NAME:-ldequipreserve}-frontend
    restart: unless-stopped
    ports:
      - "${FRONTEND_PORT:-80}:80"
    depends_on:
      - backend

volumes:
  db_data:
```
